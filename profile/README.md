# Cidade Integra ♻ – Plataforma de Denúncias Urbanas

Bem-vindo à  **Cidade Integra**, uma plataforma colaborativa voltada para a melhoria das cidades por meio da participação ativa da comunidade. Esta organização é responsável pelo desenvolvimento do projeto, que faz parte do Projeto Interdisciplinar do curso de  **Desenvolvimento de Software Multiplataforma**  da  **Fatec Matão “Luiz Marchesan”**.

Confira aqui 👉🏽 https://cidadeintegra.web.app

## 🌍 Sobre o Projeto

O  **Cidade Integra**  é uma plataforma online que permite aos cidadãos registrarem denúncias urbanas, facilitando a comunicação entre a população e as autoridades responsáveis e a resolução de problemas que impactam diretamente a qualidade de vida nas cidades. O projeto está alinhado ao  **Objetivo de Desenvolvimento Sustentável (ODS) 11 da ONU**, que visa tornar as cidades mais  **inclusivas, seguras, resilientes e sustentáveis**.

### 🚩 Objetivo

Nosso objetivo é desenvolver um site funcional que aplique os conhecimentos adquiridos ao longo do curso, promovendo boas práticas de desenvolvimento web, design e colaboração em equipe. Além disso, buscamos gerar impacto social real ao fomentar o engajamento cívico e facilitar a atuação do poder público na resolução de demandas urbanas.

## 🎯 Metodologia SMART

O desenvolvimento do projeto segue a metodologia SMART, com metas bem definidas para garantir a clareza, viabilidade e impacto da solução:

**Specific (Específico):**  Criar uma aplicação web inovadora para registro e gestão de denúncias urbanas (como lixo acumulado, vazamentos, obras irregulares etc.), promovendo a participação cidadã e a melhoria da infraestrutura urbana por meio da comunicação entre população e autoridades.

**Measurable (Mensurável):**  Investimento estimado de R$ 105.000. A conclusão está prevista para 23 de junho de 2025, com acompanhamento de tarefas e metas inspiradas em plataformas como FixMyBerlin, FixMyStreet e SeeClickFix.

**Attainable (Atingível):**  O projeto é viável dentro do prazo de 5 meses, considerando a equipe de 5 alunos, com suporte docente e uso de ferramentas como React e Firebase. O foco está em modularização, Arquitetura Limpa e testes manuais para garantir qualidade.

**Relevant (Relevante):**  Contribui diretamente para a manutenção eficiente e sustentável das cidades, fortalecendo a integração entre cidadãos e gestão pública, com impacto positivo na qualidade de vida urbana.

**Time-bound (Temporal):**  A entrega final está programada para 23 de junho de 2025, com organização baseada em cronograma de sprints, reuniões semanais e testes contínuos ao longo do desenvolvimento.

### 🎯 Qual o Problema a ser Resolvido?

Atualmente, a comunicação entre os cidadãos e os serviços públicos muitas vezes é ineficiente, descentralizada ou inacessível, dificultando a resolução de problemas urbanos cotidianos. Isso gera frustração na população, acúmulo de demandas não atendidas e degradação do espaço público.

### ✅ Solução Proposta

O  **Cidade Integra**  propõe uma  **plataforma web responsiva**  e de fácil uso, onde os cidadãos possam:

Registrar e acompanhar denúncias urbanas;

Interagir com atualizações das autoridades ou administradores do sistema.

### 🧑‍🤝‍🧑 Público-Alvo

A plataforma é voltada principalmente para:

**Cidadãos**  que desejam reportar problemas urbanos de forma prática e transparente;

**Órgãos públicos municipais**  interessados em aprimorar a gestão de demandas urbanas;

### 🚀 MVP – Produto Mínimo Viável

O MVP do Cidade Integra é uma versão funcional da plataforma que entrega as funcionalidades essenciais para validar a proposta com os usuários finais. Ele foi projetado para testar a usabilidade, identificar pontos de melhoria e comprovar o valor da solução antes de evoluções futuras

## ⚙️ Funcionalidades

📌  **Cadastro de denúncias:**  com campos para descrição, localização (CEP ou Endereço) e upload de imagens;

👥  **Cadastro de Usuários:**  com campos para nome, e-mail e senha;

🔍  **Consulta pública:**  listagem das denúncias por localidade, status ou categoria;

👤  **Sistema de autenticação:**  para usuários (cidadãos) e administradores (órgãos responsáveis);

🧭  **Painel administrativo:**  para acompanhamento, priorização e resposta das denúncias;

📝  **Histórico de atualizações**  das denúncias (timeline);

📱  **Interface responsiva**  e acessível via dispositivos móveis e desktop.

Esse modelo visa  **centralizar a comunicação**,  **organizar dados úteis para as prefeituras**  e  **fortalecer a cultura da cidadania ativa**.

### 🔹 Padrão de Commits

Utilizamos o padrão  **Conventional Commits**  para manter um histórico organizado. A estrutura básica dos commits deve seguir o formato:

```
tipo: descrição breve
```

**Tipos mais comuns:**

`feat`: Adição de nova funcionalidade

`fix`: Correção de bug

`docs`: Alterações na documentação

`style`: Mudanças que não afetam a lógica (espaços, formatação)

`refactor`: Refatoração de código

`test`: Adição ou modificação de testes

`chore`: Tarefas auxiliares (build, dependências, etc.)

Exemplo:

```
feat: adicionar formulário de denúncia
```

### 🔹 Como Enviar um Pull Request (PR)

1. **Crie um Fork**  do repositório.

2. **Crie uma nova branch**  descritiva:

3. git checkout -b feature/nome-da-feature

4. **Faça as alterações e commits**  seguindo o padrão.

5. **Envie as alterações para seu Fork:**

6. git push origin feature/nome-da-feature

7. **Abra um Pull Request**  no repositório principal e descreva suas alterações.

8. Aguarde enquanto revisamos a sua versão 😀
